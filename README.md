# 会議作業管理

`SPEC.md` に基づく、個人用の Google Apps Script Web アプリです。文字起こしが届いた後の会議解析と理解度クイズを自動生成し、全問正解した会議ごとに作業ガイド初稿、AIによる自己検収・修正、承認待ち保存までを自動化します。利用者は主に「クイズへ回答」と「ガイドを確認してOK」を行い、必要な場合だけ修正点をAIへ返します。

## 自動セットアップ・デプロイ

UIを Playwright で操作する方式ではなく、Google 公式の `clasp` を使います。Apps Script の画面変更に影響されにくく、ログイン情報や既存ブラウザ Cookie をスクリプトへ渡す必要がありません。

前提は Node.js 20 以上と、Google アカウント側の [Apps Script API 有効化](https://script.google.com/home/usersettings) です。CLI の詳細は [Google 公式 clasp ガイド](https://developers.google.com/apps-script/guides/clasp) を参照してください。

```bash
npm install
npm run setup
```

初回は次の処理が順番に行われます。

1. 未ログインなら認証手順を端末に表示して `clasp login` を起動する（Google の画面で本人がログイン）。認証完了後はログイン状態を再確認してから続行する。
2. Apps Script API が無効なら設定ページを開き、ユーザーの有効化を待つ。
3. 本人所有の Web アプリ用 Apps Script プロジェクトを作成する。
4. リポジトリの `.gs` / `.html` / `appsscript.json` を push する。
5. 本人限定 Web アプリとして versioned deployment を作成する。
6. `?setup=1` 付きURLをブラウザで開く。初回の権限同意後、Driveフォルダと管理スプレッドシートを自動作成する。

ブラウザを開けない環境ではURLが端末に表示されます。手動で開いてログイン・同意してください。

既存 Apps Script プロジェクトを使う場合:

```bash
npm run setup -- --script-id YOUR_SCRIPT_ID
```

二回目以降の更新デプロイ:

```bash
npm run deploy
```

CI やヘッドレス環境では `--no-open` を指定できます。未認証の場合は端末に認証URLが表示されるので、ブラウザで認証後、アドレスバーのURL全体を端末へ貼り付けてください。`.clasp.json` と `.deploy-state.json` はローカル環境情報として Git 管理から除外されます。`~/.clasprc.json` は OAuth 更新トークンを含むため、共有・commitしないでください。

## トップページでの文字起こし選択と完了管理

ダッシュボード（トップページ）に「文字起こしドキュメント」パネルがあり、`01_文字起こし` フォルダ内の Google ドキュメントを一覧できます。

- 各ドキュメントには進捗（未登録 / 解析待ち / クイズ未回答 / ガイド作成・実行）が表示され、選択すると次の工程の画面へそのまま移動します。
- クイズ回答やガイド作成などが終わったドキュメントは「完了にする」で `01_文字起こし/完了` フォルダへ移動し、完了扱いになります（対応する会議レコードにも完了日時を記録）。
- 完了済み一覧から「未完了に戻す」を押すと、元のカテゴリーフォルダへ戻せます。

既存環境で更新する場合は、デプロイ後に `?setup=1` を一度開いてください。不足フォルダ・管理シート列・`AiInteractions` シートが追加され、既存の自動登録トリガーは5分間隔へ更新されます。既存データは削除されません。

## JSONを作る前の認識合わせ

コピー＆ペーストで生成AIを使う手動経路は、最初からアプリ用JSONを要求しません。会議解析と作業ガイドで、それぞれ人が根拠と不足を確認してからJSON化します。

1. 会議解析では、生成AIに読みやすい「会議内容の確認案」を作らせます。要約、決定・未決事項、クイズの出題方向、作成候補ガイドを確認します。ガイド候補には「何をするガイドか」「完了状態」「必要な具体作業」「事前確認」を含めます。
2. 人が確認案を直接修正し、方向性を承認した後にだけ、その確認済み内容と文字起こしを含むJSON生成プロンプトを作れます。
3. 作業ガイドでは、文字起こしから骨格と知識台帳を作り、AIの質問に回答して不足事実を集めます。
4. 準備度ゲートで未確認項目を確認または明示的に降格した後にだけ、確認済み台帳からJSON生成プロンプトを作れます。

## 品質基準と、達するまでの生成AIとの往復

作業ガイドの目標品質は、本番環境のセットアップ手順書と同じ具体度です。すべてのガイド系プロンプトに共通の品質基準（開く画面のURL、画面表記どおりのボタン・メニュー名、入力値の形式と例、客観的に判定できる完了確認、失敗時の対処、前提・注意の網羅）が埋め込まれます。一度の生成でこの水準に届かなくても、次の工程で不足を事実として埋めます。

- 取材ループ（STEP 5）: AIが最大5問を提示し、本人が選択式の回答やURL・画面文言・コマンド出力などの現物を返します。台帳更新・矛盾・仮値・準備度を毎回同じ形式で確認できます。
- 机上実行レビュー（STEP 8）: 別チャットの初見の作業者として上から実行させ、詰まりを台帳の `unknown` へ戻します。解消後に再生成するか、「作業前に確認」へ明示的に降格します。
- 具体度チェック（STEP 8）: 文字数・URL・完了条件の補助チェックも残し、修正版JSONは STEP 7 へ貼り直して再度台帳突合します。

作業ガイドの深掘りは、文章の書き直しではなく「知識台帳」を介した取材ループで行います。文字起こしから骨格と事実を抽出し、URL・画面表記・入力値・完了確認・失敗時対処などを `confirmed`（確認済み）/ `assumed`（仮値）/ `unknown`（不明）で管理します。準備度ゲートではステップ×スロットのヒートマップを表示し、すべてを確認済みにするか、本人が「作業前に確認」へ明示的に降格してからガイドを生成します。

サーバー側でも、準備度ゲートを通っていない新規ガイドJSONの取り込み、机上実行レビューを完走していない手動ガイドの保存、台帳にない具体値を含む編集後ガイドの保存を拒否します。主要なAI往復は従来どおりAI対話履歴へ保存されます。

「管理・診断」で明示的に有効化するAPI全自動処理も同じ台帳モデルを使います。人へ質問できないため `unknown` / `assumed` は自動で「作業前に確認」へ降格し、初見の作業者ロールによる机上実行レビューの詰まりも前提条件と残存リスクへ記録して、承認待ちで保存します。

## 文字起こしから承認待ちガイドまでの全自動処理

「管理・診断」でプロバイダー（OpenAI / Google Gemini / Anthropic Claude / OpenRouter）、APIキー、利用モデル、データ送信確認を設定し、「解析・クイズ・作業ガイド生成・AI検収を全自動で行う」を有効にします。その後「文字起こしからの全自動処理」を有効にすると、5分ごとの時間主導トリガーが次を行います。

1. `01_文字起こし` の未登録ファイルを会議として登録する。
2. 会議記録を解析し、会議要約・決定事項・未決事項・作業候補・理解度クイズを生成して保存する。
3. 利用者が会議ごとの理解度クイズへ回答する。全問正解した会議だけが次へ進む。
4. 本人が実行・確認・委譲する具体作業について、文字起こしから骨格と知識台帳を初期化する。未確認項目は「作業前に確認」へ自動降格する。
5. 準備度ゲート通過後、確認済み台帳だけを事実源として作業ガイドを組版し、台帳にないURL・主要な具体値・根拠参照を機械検証する。
6. 別のAI検収工程と、初見の作業者ロールによる机上実行レビューを行う。詰まりは前提条件と残存リスクへ明示する。
7. GoogleドキュメントとJSONを `03_作業ガイド/要確認` に保存する。
8. 利用者が作業ガイド詳細の「このガイドでOK」を押すと、2ファイルを `実行可能` へ移動する。

「今すぐ実行」でも同じキューを処理できます。キューは会議単位で独立しているため、ある会議がクイズ待ちでも、全問正解済みの別会議はガイド生成を進められます。同時に起動したキューは短時間の取得ロックで別の候補を確保し、AI生成中は互いを待ちません。GASの実行時間上限へ近づく場合は状態を保存して止まり、残りを次のトリガーへ引き継ぎます。APIエラーやJSON検証失敗は理由とともに管理画面へ表示され、同一項目の無制限な再試行は行いません。

生成AI接続はプロバイダーごとのアダプター（`AiProviderAdapters.gs`）を介して行い、呼び出し側のコードはプロバイダー名を意識しません。OpenAIは Responses API のJSONモード、Google Geminiは `generateContent`（`responseMimeType: application/json`）、Anthropic ClaudeはMessages API、OpenRouterはOpenAI互換のChat Completions APIを使用し、いずれもアプリ側の既存スキーマ検証にも通った回答だけを保存します。新しいプロバイダーを追加する場合も `AiProviderAdapters.gs` にリクエスト組み立て・レスポンス解析・エラー整形の3関数を追加するだけで済みます。APIキーは Script Properties に保存し、ブラウザへ再表示しません。APIへ送ったプロンプトと回答、検証結果、自動修正の各往復は `02_会議解析/AI対話履歴` と `AiInteractions` シートに保存され、クイズ画面・作業ガイド詳細から後で確認できます。OpenAIの場合、APIリクエストは `store: false` で送信します。

手動作成は自動化設定の有無にかかわらず常時利用できます。メインメニューの「ガイド作成」または作業ガイド画面の「手動で新規作成」から作業候補を選び、P1（骨格・台帳初期化）→ P2（最大5問の取材ラウンド）→ 準備度ゲート → P3（台帳からのJSON組版）→ P4（別チャットでの机上実行レビュー）→ 編集・保存の順に進めます。P2ではURL、画面の文言、コマンド出力などの現物を貼り付けるほど、複数の穴を一度に確認済みにできます。作成途中でガイドが不要だと判断した場合は「このガイドは不要」で作成を中止し、候補を手動・全自動のガイド対象から削除できます。API自動生成と手動作成は併用でき、手動で作成中の候補をバックグラウンドの自動生成が処理することはありません。手動の主要なAI往復と台帳更新監査も同じ履歴へ保存されます。

実行画面の各手順には「ガイド外の資料を見た / 人に聞いた」ボタンがあります。何を調べたかを1行で記録すると、次バージョン作成時の取材対象として引き継がれます。

### 承認と修正

- `承認待ち` のガイドは実行開始できません。「このガイドでOK」で承認したバージョンだけが実行可能です。
- 修正したい場合はガイド詳細へ指摘だけ入力します。AIが修正版を生成し、もう一度自動検収して新バージョンを `承認待ち` にします。
- AIが任意コードを生成・実行することはありません。ガイド内の `script` 手順に使えるのは `AllowedActionRegistry` へ本人が登録した既存処理だけです。

### スクリプト化候補の自動検出

「管理・診断」画面の「スクリプト化候補（繰り返し検出）」に、複数の作業ガイドまたは複数回の実行で繰り返されている手動（input）手順が表示されます。手順名を正規化して同一視し、繰り返し回数の多い順に提示するので、上位の候補から登録スクリプト化を検討できます（SPEC の最終目的「繰り返し作業の自動化スクリプト蓄積」の起点になります）。

### 登録スクリプトライブラリ

作業ガイドの `script` 手順から実行できる実装済みアクションを拡充しています。いずれも「管理・診断」画面で本人が明示登録したものだけが実行可能です。

| スクリプトID | 内容 |
|---|---|
| `UTILITY_TIMESTAMP` | 現在日時を記録 |
| `DRIVE_COPY_FILE` | Drive ファイルを複製 |
| `DRIVE_MOVE_FILE` | Drive ファイルを移動 |
| `DRIVE_CREATE_FOLDER` | Drive フォルダを作成 |
| `DOC_CREATE_TEXT` | Google ドキュメントを新規作成 |
| `SHEET_APPEND_ROW` | スプレッドシートへ1行追記 |
| `GMAIL_CREATE_DRAFT` | Gmail 下書きを作成（自動送信はしない） |
| `CALENDAR_CREATE_EVENT` | カレンダー予定を作成 |

Gmail・カレンダーを使うアクションを初めて実行する際は、Google の権限承認が追加で求められます。

## 手動セットアップ（代替）

`clasp` を使えない環境向けに、貼り付け用の [Code.gs](dist/Code.gs) と [Index.html](dist/Index.html) を用意しています。Apps Script エディタではこの2ファイルを一回ずつ貼り付ければよく、分割された各ソースを個別に作成する必要はありません。詳しい手順は [SETUP_GUIDE.md](SETUP_GUIDE.md) を参照してください。

`dist/Code.gs` では、利用者が変更するフォルダ名・スプレッドシート名・分類などを含む `Config.gs` セクションを先頭に配置しています。設定を変更する場合は、生成物を直接編集せず、リポジトリ直下の `Config.gs` を編集してから再生成してください。

分割ソースを更新した場合は、Node.js 20以上の環境で次を実行すると2ファイルを再生成できます。この処理自体に `clasp` は不要です。

```bash
npm run build
```

`dist/**` は `.claspignore` の対象です。`clasp` が使える環境の `clasp push` / `npm run setup` / `npm run deploy` では、貼り付け用の連結コードは送信されず、ルートにある分割ソースだけが反映されます。Webアプリが開けていれば、未設定時のダッシュボードにある「初期設定を実行」でも初期設定できます。

## セキュリティ

- `appsscript.json` の Web アプリ設定は `MYSELF` / `USER_DEPLOYING` です。
- Drive 操作はブラウザから直接行わず、すべて `google.script.run` 経由です。
- 作業ガイドの `script` 手順は、コード実装済みかつ `AllowedActionRegistry` へ本人が登録したIDだけを実行します。
- AI回答は任意コードとして実行せず、JSONとしてスキーマ検証します。
- 生成AI APIキー（OpenAI / Gemini / Anthropic / OpenRouter共通）はScript Propertiesに保存し、レスポンスや監査ログへ含めません。
- 全自動処理を有効にする前に、会議記録・選択資料を外部AI APIへ送ることを画面で明示確認します。
