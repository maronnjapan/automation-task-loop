# 会議作業管理

`SPEC.md` に基づく、個人用の Google Apps Script Web アプリです。会議文字起こしの解析、理解度クイズ、必要な作業候補だけのガイド化、Googleドキュメント/JSON保存、中断・再開可能な実行までを一つの画面で扱います。

## 自動セットアップ・デプロイ

UIを Playwright で操作する方式ではなく、Google 公式の `clasp` を使います。Apps Script の画面変更に影響されにくく、ログイン情報や既存ブラウザ Cookie をスクリプトへ渡す必要がありません。

前提は Node.js 20 以上と、Google アカウント側の [Apps Script API 有効化](https://script.google.com/home/usersettings) です。CLI の詳細は [Google 公式 clasp ガイド](https://developers.google.com/apps-script/guides/clasp) を参照してください。

```bash
npm install
npm run setup
```

初回は次の処理が順番に行われます。

1. 未ログインなら認証手順を端末に表示して `clasp login` を起動する（Google の画面で本人がログイン）。認証完了後はログイン状態を再確認してから続行する。
2. Apps Script API が無効なら設定ページを開き、ユーザーの有効化を待つ。
3. 本人所有の Web アプリ用 Apps Script プロジェクトを作成する。
4. リポジトリの `.gs` / `.html` / `appsscript.json` を push する。
5. 本人限定 Web アプリとして versioned deployment を作成する。
6. `?setup=1` 付きURLをブラウザで開く。初回の権限同意後、Driveフォルダと管理スプレッドシートを自動作成する。

ブラウザを開けない環境ではURLが端末に表示されます。手動で開いてログイン・同意してください。

既存 Apps Script プロジェクトを使う場合:

```bash
npm run setup -- --script-id YOUR_SCRIPT_ID
```

二回目以降の更新デプロイ:

```bash
npm run deploy
```

CI やヘッドレス環境では `--no-open` を指定できます。未認証の場合は端末に認証URLが表示されるので、ブラウザで認証後、アドレスバーのURL全体を端末へ貼り付けてください。`.clasp.json` と `.deploy-state.json` はローカル環境情報として Git 管理から除外されます。`~/.clasprc.json` は OAuth 更新トークンを含むため、共有・commitしないでください。

## トップページでの文字起こし選択と完了管理

ダッシュボード（トップページ）に「文字起こしドキュメント」パネルがあり、`01_文字起こし` フォルダ内の Google ドキュメントを一覧できます。

- 各ドキュメントには進捗（未登録 / 解析待ち / クイズ未回答 / ガイド作成・実行）が表示され、選択すると次の工程の画面へそのまま移動します。
- クイズ回答やガイド作成などが終わったドキュメントは「完了にする」で `01_文字起こし/完了` フォルダへ移動し、完了扱いになります（対応する会議レコードにも完了日時を記録）。
- 完了済み一覧から「未完了に戻す」を押すと、元のカテゴリーフォルダへ戻せます。

既存環境で更新する場合は、デプロイ後にダッシュボードの「初期設定を実行」（または `?setup=1`）を一度実行してください。`完了` フォルダと Meetings シートの `workflowStatus` / `completedAt` 列が自動追加されます。

## 作業の自動化（生成AI API不使用）

繰り返し作業を GAS 内で完結する自動化に置き換えるための機能です。従量課金の生成AI APIは一切使用しません（AI連携は従来どおりコピー&ペーストのみ）。

### 文字起こしの自動会議登録

「管理・診断」画面の「文字起こしの自動登録」で有効化すると、時間主導トリガー（30分ごと）が `01_文字起こし` フォルダを確認し、未登録の文字起こしを会議として自動登録します。

- タイトルはファイル名、カテゴリーは格納フォルダから自動判定します（手動登録と同じ規則）。
- 登録後はダッシュボードに「解析待ち」として表示され、解析プロンプトの作成へそのまま進めます。
- 「今すぐ実行」でトリガーを待たずに同じ処理を実行できます。読み取りできないファイル等は理由付きで「対象外」として最終実行結果に表示されます。
- トリガーの有効化には Apps Script の権限承認（スクリプトトリガー管理）が追加で必要になる場合があります。

### スクリプト化候補の自動検出

「管理・診断」画面の「スクリプト化候補（繰り返し検出）」に、複数の作業ガイドまたは複数回の実行で繰り返されている手動（input）手順が表示されます。手順名を正規化して同一視し、繰り返し回数の多い順に提示するので、上位の候補から登録スクリプト化を検討できます（SPEC の最終目的「繰り返し作業の自動化スクリプト蓄積」の起点になります）。

### 登録スクリプトライブラリ

作業ガイドの `script` 手順から実行できる実装済みアクションを拡充しています。いずれも「管理・診断」画面で本人が明示登録したものだけが実行可能です。

| スクリプトID | 内容 |
|---|---|
| `UTILITY_TIMESTAMP` | 現在日時を記録 |
| `DRIVE_COPY_FILE` | Drive ファイルを複製 |
| `DRIVE_MOVE_FILE` | Drive ファイルを移動 |
| `DRIVE_CREATE_FOLDER` | Drive フォルダを作成 |
| `DOC_CREATE_TEXT` | Google ドキュメントを新規作成 |
| `SHEET_APPEND_ROW` | スプレッドシートへ1行追記 |
| `GMAIL_CREATE_DRAFT` | Gmail 下書きを作成（自動送信はしない） |
| `CALENDAR_CREATE_EVENT` | カレンダー予定を作成 |

Gmail・カレンダーを使うアクションを初めて実行する際は、Google の権限承認が追加で求められます。

## 手動セットアップ（代替）

自動スクリプトを使えない場合は [SETUP_GUIDE.md](SETUP_GUIDE.md) に従ってください。Webアプリが開けていれば、未設定時のダッシュボードにある「初期設定を実行」でも同じ処理を行えます。

## セキュリティ

- `appsscript.json` の Web アプリ設定は `MYSELF` / `USER_DEPLOYING` です。
- Drive 操作はブラウザから直接行わず、すべて `google.script.run` 経由です。
- 作業ガイドの `script` 手順は、コード実装済みかつ `AllowedActionRegistry` へ本人が登録したIDだけを実行します。
- AI回答は任意コードとして実行せず、JSONとしてスキーマ検証します。
