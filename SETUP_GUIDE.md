# セットアップガイド

このプロジェクトの `SetupGuide.gs` が、`SPEC.md` に定義された Drive フォルダ、管理スプレッドシート、全管理シートを一括作成します。既存データは削除せず、不足分だけを補うため再実行できます。

通常は `npm install && npm run setup` でプロジェクト作成・push・deploy・この初期設定まで自動化できます。`clasp` を使える場合はこの方法を使用してください。`.claspignore` で `dist/**` を除外しているため、貼り付け用の連結コードが分割ソースと重複して push されることはありません。

## `clasp` を使えない場合

リポジトリに生成済みの `dist/Code.gs` と `dist/Index.html` があるため、コードの貼り付けは二回だけです。分割ソースを変更した後に生成物を更新する場合は、Node.js 20以上の環境で `npm run build` を実行してください（`clasp` は不要です）。

1. Apps Script で新しいスタンドアロンプロジェクトを作成する。
2. 最初からある `Code.gs` の内容をすべて削除し、`dist/Code.gs` の内容を一回貼り付ける。フォルダ名など利用者が変更する設定は、貼り付けたコードの先頭にある `Config.gs` セクションへまとまっている。
3. 左側の「ファイルを追加 > HTML」を選び、名前を拡張子なしで `Index` として作成する。作成された `Index.html` の内容をすべて削除し、`dist/Index.html` の内容を一回貼り付ける。
4. 左側の「サービスを追加」から `Drive API`（v3、識別子 `Drive`）を追加する。
5. Apps Script エディタで `runSetupGuide()` を実行し、Drive / Docs / Sheets の権限を許可する。
6. 戻り値の `diagnostics` がすべて `ok` であることを確認する。必要なら `runSetupDiagnostics()` を単独実行する。
7. 「デプロイ > 新しいデプロイ > ウェブアプリ」を選び、「次のユーザーとして実行: 自分」「アクセスできるユーザー: 自分のみ」で公開する。
8. Web アプリの「管理・診断 > 生成AI接続」でプロバイダー（OpenAI / Google Gemini / Anthropic Claude / OpenRouter）、APIキー、利用モデル、JSON自動修復回数を設定する。
9. 会議記録・参照資料をAPIへ送ることを確認し、「解析・クイズ・作業ガイド生成・AI検収を全自動で行う」を有効にする。
10. 「接続テスト」が成功することを確認する。
11. 「文字起こしからの全自動処理」を有効にする。Apps Scriptからトリガー管理と外部リクエストの追加権限を求められた場合は内容を確認して許可する。
12. `01_文字起こし` へテスト用のGoogleドキュメントを置き、「今すぐ実行」でクイズと承認待ちガイドが作られることを確認する。

APIを使わない場合は手順8〜12を省略できます。メインメニューの「ガイド作成」または作業ガイド画面の「手動で新規作成」から、プロンプトのコピー＆回答JSONの貼り付けによる手動作成を利用してください。手動作成はAPI自動化を設定した後も常時利用できます。

`clasp` を使う場合は `appsscript.json` にも本人限定設定を含めています。手動の場合を含め、デプロイ時の画面でも公開範囲を必ず確認してください。
