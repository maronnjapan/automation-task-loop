# セットアップガイド

このプロジェクトの `SetupGuide.gs` が、`SPEC.md` に定義された Drive フォルダ、管理スプレッドシート、全管理シートを一括作成します。既存データは削除せず、不足分だけを補うため再実行できます。

通常は `npm install && npm run setup` でプロジェクト作成・push・deploy・この初期設定まで自動化できます。以下は自動スクリプトを使えない場合の代替手順です。

1. このディレクトリを `clasp push` などで Apps Script プロジェクトへ配置する。
2. Apps Script エディタで `runSetupGuide()` を実行し、Drive / Docs / Sheets の権限を許可する。
3. 戻り値の `diagnostics` がすべて `ok` であることを確認する。必要なら `runSetupDiagnostics()` を単独実行する。
4. 「デプロイ > 新しいデプロイ > ウェブアプリ」を選び、「次のユーザーとして実行: 自分」「アクセスできるユーザー: 自分のみ」で公開する。
5. Web アプリの「管理・診断 > 生成AI接続」でプロバイダー（OpenAI / Google Gemini / Anthropic Claude / OpenRouter）、APIキー、利用モデル、JSON自動修復回数を設定する。
6. 会議記録・参照資料をAPIへ送ることを確認し、「解析・クイズ・作業ガイド生成・AI検収を全自動で行う」を有効にする。
7. 「接続テスト」が成功することを確認する。
8. 「文字起こしからの全自動処理」を有効にする。Apps Scriptからトリガー管理と外部リクエストの追加権限を求められた場合は内容を確認して許可する。
9. `01_文字起こし` へテスト用のGoogleドキュメントを置き、「今すぐ実行」でクイズと承認待ちガイドが作られることを確認する。

APIを使わない場合は手順5〜9を省略できます。メインメニューの「ガイド作成」または作業ガイド画面の「手動で新規作成」から、プロンプトのコピー＆回答JSONの貼り付けによる手動作成を利用してください。手動作成はAPI自動化を設定した後も常時利用できます。

`appsscript.json` にも本人限定設定を含めていますが、デプロイ時の画面でも公開範囲を必ず確認してください。
